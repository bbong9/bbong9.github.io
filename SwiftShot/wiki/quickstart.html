入门

快速开始

从 GitHub Releases 下载、安装、授权到第一次截图，按最短路径验证 SwiftShot 是否可用。

适用场景

  • 第一次在 macOS 上安装 SwiftShot。
  • 需要快速验证菜单栏入口、截图 Overlay、编辑器和复制保存链路。
  • 需要给真实设备 QA 准备最小操作流程。

前置条件

  • macOS 13 或更高版本。
  • 能访问 SwiftShot 的 GitHub Releases 页面。
  • 首次截图前需要在系统设置中授予屏幕录制权限；窗口识别、长截图等高级能力还需要辅助功能权限。

下载并启动

  1. 打开 SwiftShot GitHub Releases。
  2. 下载最新版本的 SwiftShot.dmg。
  3. 打开 DMG，把 SwiftShot.app 拖到 Applications。
  4. 从应用程序启动 SwiftShot，确认菜单栏出现 SwiftShot 图标。

普通用户默认走 Release 下载，不需要 clone 仓库，也不需要本地构建。

第一次截图

  1. 启动后确认顶部菜单栏出现 SwiftShot 图标。
  2. 按当前设置页显示的全局快捷键，默认项目曾使用 Command + Shift + 2，当前本机配置以设置页“快捷键”为准。
  3. 拖拽框选区域，或移动鼠标让窗口高亮。
  4. 按 Enter 或点击复制按钮完成；按 Esc 取消。
  5. 如果使用编辑器入口，确认图片进入 SwiftShot 编辑器，工具栏位于截图内容外侧。

验证方式

  • 剪贴板能粘贴刚截取的图片。
  • 开启自动保存后，默认目录出现截图文件。
  • 无权限时能看到明确系统设置入口，而不是静默失败或崩溃。
  • 截图完成后不包含 Overlay 蓝框、工具栏、控制点或尺寸浮标。

风险提示

  • 如果下载包第一次打开被系统拦截，请在系统设置的安全提示中确认来源后再打开。
  • 不要把含有真实聊天、密钥、客户数据或个人隐私的截图放进公开 Wiki。
  • 混合 DPI 多屏仍需要真实设备验证像素级坐标。
